**Accent reserved for:**
1. FAB background and icon
2. Selected tab indicator
3. Primary CTA button background (`ElevatedButton`)
4. Segmented button selected state
5. Form field focus border
6. Active filter chip
7. Brand glassmorphism shadow tint (light mode only — dark mode uses black shadow)

### Dark Theme Elevation Strategy (per D-09, D-11)

Material Design dark theme uses **surface tint** to indicate elevation. For this phase:

| Surface Level | Light Mode | Dark Mode | Alpha |
|---------------|-----------|-----------|-------|
| Base (scaffold) | `#F8F6F6` | `#221610` | — |
| Flat (cards) | `#FFFFFF` | `#1A1210` | 0.85 (glass) |
| Elevated (modals) | `#FFFFFF` | `#1A1210` | 0.90 (glass overlay) |
| Glass Navigation | `#FFFFFF` | `#1A1210` | 0.90 (glass nav) |

**WCAG Contrast Requirements** (per THM-06, D-10):
- Normal text: ≥ 4.5:1 contrast ratio
- Large text (≥ 18px / 14px bold): ≥ 3:1 contrast ratio
- Icons: ≥ 3:1 contrast ratio
- Verification: Manual with contrast checker tool on key color pairs

### Glassmorphism Dark Mode Redesign (per D-09)

This is the **primary visual design challenge** of Phase 1. Current dark glass values are starting points, not final.

| Property | Light Mode | Dark Mode (Current) | Dark Mode (Target) |
|----------|-----------|--------------------|--------------------|
| Glass surface alpha | 0.95 | 0.92 | Needs redesign — verify readability |
| Glass card alpha | 0.85 | 0.82 | Needs redesign — verify contrast |
| Glass pill alpha | 0.70 | 0.65 | Needs redesign — verify visibility |
| Glass nav alpha | 0.90 | 0.88 | Needs redesign — verify tab visibility |
| Glass border (white) | 0.10 | 0.08 | Needs redesign — may need increase |
| Glass blur (cards) | 12px | 13.2px (1.1×) | Needs redesign — may need more blur |
| Glass shadow | Orange-tinted 0.08 | Black 0.15 | Needs redesign — verify depth perception |

**Redesign priorities** (from D-08 "no shortcuts"):
1. Bottom navigation — tabs must be clearly readable
2. Transaction cards — primary content surface
3. Glass form surfaces — input fields need contrast
4. Glass overlays / bottom sheets — modal readability
5. Glass pills / chips — category indicators visibility

### Grep Targets for Hardcoded Colors

Search patterns to find violations:
- `Colors.white` — should use `AppColors.getSurfaceColor(isDark)` or `Theme.of(context).colorScheme.surface`
- `Colors.black` — should use `AppColors.textPrimary` or theme text color
- `Colors.grey` — should use `AppColors.textSecondary`/`textTertiary` or `Colors.grey.shadeXXX` with theme awareness
- `Color(0xFF...)` — should use `AppColors` constants or theme `ColorScheme`
- `Colors.transparent` — acceptable, no change needed

## Accessibility Contract (per THM-06, D-10)

| Element Type | Min Ratio | Verification Method |
|-------------|-----------|-------------------|
| Normal text (< 18px) | 4.5:1 | Manual contrast checker tool |
| Large text (≥ 18px or ≥ 14px bold) | 3:1 | Manual contrast checker tool |
| Icons | 3:1 | Manual contrast checker tool |
| UI components (borders, focus) | 3:1 | Manual contrast checker tool |

### Key Contrast Pairs to Verify

| Pair | Light Mode | Dark Mode | Target |
|------|-----------|-----------|--------|
| Body text on surface | `#1E293B` on `#FFFFFF` | `#F8F6F6` on `#1A1210` | ≥ 4.5:1 |
| Secondary text on surface | `#64748B` on `#FFFFFF` | `#64748B` on `#1A1210` | ≥ 4.5:1 (dark may fail!) |
| Accent on white | `#EC5B13` on `#FFFFFF` | N/A | ≥ 3:1 (for large text/icons) |

| Tab icon (unselected) | `#64748B` on `#FFFFFF` | `#64748B` on glass | ≥ 3:1 |

**Note:** `#64748B` on `#1A1210` = ~4.6:1 (passes). Verify in glassmorphism context where alpha reduces effective contrast.
