**Step 2 — Add alert check method to `_TransactionFormScreenState`:**

Add a private method following the exact pattern from transaction_list_screen.dart lines 641-683:
```dart
/// Check budget alerts after a transaction is saved per BUD-04
/// Must not block transaction flow — errors are caught and logged
Future<void> _checkBudgetAlertAfterSave(TransactionEntity transaction) async {
  try {
    final controller = ref.read(budgetAlertControllerProvider);
    final alertType = await controller.checkAlertsAfterTransaction(
      categoryId: transaction.categoryId,
      year: transaction.dateTime.year,
      month: transaction.dateTime.month,
    );

    if (!mounted) return;
    if (alertType == BudgetAlertType.none) return;

    // Get category name for alert message
    final categoriesAsync = ref.read(categoryListProvider);
    String categoryName = 'Kategori';
    categoriesAsync.whenData((categories) {
      final cat = categories.where((c) => c.id == transaction.categoryId).firstOrNull;
      if (cat != null) categoryName = cat.name;
    });

    final message = switch (alertType) {
      BudgetAlertType.warning => '⚠️ Anggaran $categoryName sudah 75% terpakai',
      BudgetAlertType.limit => '🔴 Anggaran $categoryName sudah mencapai batas',
      BudgetAlertType.over => '🚨 Anggaran $categoryName melebihi batas!',
      BudgetAlertType.none => '',
    };

    if (message.isEmpty) return;

    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(
        content: Text(message),
        duration: const Duration(seconds: 4),
        behavior: SnackBarBehavior.floating,
      ),
    );
  } catch (e, stackTrace) {
    // Must not block transaction flow
    AppLogger.e('BudgetAlert: Error after transaction save', e, stackTrace);
  }
}
```

**Step 3 — Call alert check in the save flow:**

In `_buildSubmitButton` (around line 408-429), after the successful save and BEFORE `context.pop()`:

```dart
if (success && mounted) {
  ref.read(receiptScanProvider.notifier).reset();

  // Get the saved transaction for alert check
  final formState = ref.read(transactionFormProvider);
  final savedTransaction = formState.editingTransaction ?? 
      // For new transactions, construct from form state
      TransactionEntity(
        categoryId: formState.selectedCategoryId!,
        amount: formState.amount,
        dateTime: formState.selectedDate,
        // ... other required fields
      );

  // Show success snackbar
  ScaffoldMessenger.of(context).showSnackBar(...);

  // Check budget alerts (non-blocking, fire-and-forget)
  _checkBudgetAlertAfterSave(savedTransaction);

  context.pop();
}
```

IMPORTANT: The alert check is fire-and-forget (not awaited before pop). The `context.pop()` should NOT be delayed by the alert check. The SnackBar will show on the previous screen (TransactionListScreen) because ScaffoldMessenger is scoped to the app-level. Call the alert check and immediately pop:
```dart
// Fire-and-forget: don't await
_checkBudgetAlertAfterSave(savedTransaction);
context.pop();
```

However, if accessing the saved transaction's data from `transactionFormProvider` state is complex (the provider may have already cleared), an alternative approach is to pass the form data before pop:

1. Store the relevant data (categoryId, date) in local variables before calling submit()
2. After success, use those local variables to call the alert check

The simplest approach: capture `categoryId` and `selectedDate` from the form state BEFORE submit, then use them in the alert check after success:
```dart
// Capture before submit
final formState = ref.read(transactionFormProvider);
final categoryId = formState.selectedCategoryId;
final transactionDate = formState.selectedDate;

final success = await ref.read(transactionFormProvider.notifier).submit();

if (success && mounted) {
  ref.read(receiptScanProvider.notifier).reset();

  ScaffoldMessenger.of(context).showSnackBar(
    SnackBar(
      content: Text(formState.isEditMode ? '...' : '...'),
      backgroundColor: AppColors.success,
      behavior: SnackBarBehavior.floating,
    ),
  );

  // Check budget alerts for expense transactions only (no budgets for income)
  if (categoryId != null && formState.transactionType == TransactionType.expense) {
    _checkBudgetAlertWith(categoryId: categoryId, date: transactionDate);
  }

  context.pop();
}
```

Create a helper `_checkBudgetAlertWith({required int categoryId, required DateTime date})` that does the same thing as `_checkBudgetAlertAfterSave` but takes direct parameters instead of a TransactionEntity.
