  <files>lib/presentation/utils/error/error_message_mapper.dart</files>
  <action>
The root cause of test 5: `ErrorMessageMapper.getUserMessage()` checks for `DatabaseException` (line 34) but the budget write repository returns `DatabaseFailure` (a domain Failure subclass). The `DatabaseFailure` has a good Indonesian message ("Budget untuk kategori ini sudah ada bulan ini") but the mapper skips it and falls through to string pattern matching, eventually hitting the generic fallback.

Fix: Add a `Failure` type check BEFORE the string pattern matching. Insert this block right after the `DatabaseException` check (after line 36):

```dart
// Handle Failure types from the domain layer — they already have user-friendly messages
if (error is Failure) {
  return error.message;
}
```

This works because:
1. All Failure subclasses have a `message` field with Indonesian user-friendly text
2. The budget write repository returns `DatabaseFailure('Budget untuk kategori ini sudah ada bulan ini')` for UNIQUE violations
3. The budget write repository returns `ValidationFailure('Anggaran hanya bisa dibuat untuk kategori pengeluaran')` for expense-only constraint
4. NotFoundFailure messages are also Indonesian and user-friendly

The `DatabaseException` check stays above this — it handles raw sqflite exceptions from the database driver directly. The `Failure` check handles the domain-layer wrapped failures.

Do NOT remove any existing checks below. The `Failure` check is early-return — if error IS a Failure, we return its message immediately. All the existing string-based checks remain as fallback for non-Failure errors.

Future<void> _showDeleteConfirmation(BudgetWithSpentEntity budgetWithSpent) async {

    context: context,
    builder: (context) => AlertDialog(
      title: const Text('Hapus Anggaran'),
      content: Text('Apakah Anda yakin ingin menghapus anggaran ini?'),
      actions: [

          onPressed: () => Navigator.of(context).pop(false),
          child: const Text('Batal'),
        ),

          onPressed: () => Navigator.of(context).pop(true),
          style: TextButton.styleFrom(

          ),
          child: const Text('Hapus'),
        ),
      ],
    ),
  );

  if (confirmed != true || !mounted) return;

  final controller = ref.read(budgetFormControllerProvider);
  final result = await controller.submitDelete(budgetWithSpent.budget.id!);

  if (!mounted) return;

  if (result.isSuccess) {

        behavior: SnackBarBehavior.floating,
      ),
    );
  }
}
