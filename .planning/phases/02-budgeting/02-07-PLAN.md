
This goes in the same section as `appInitializationProvider`, after line 96.

**Step 2: Update HomeNavigationShell in `app_router.dart`**

In `_buildSeamlessBottomNav()` method (around line 341), change the `onTap` callback to also update the provider:

Current:
```dart
onTap: (index) => navigationShell.goBranch(index),
```

Change to:
```dart
onTap: (index) {
  ref.read(activeTabIndexProvider.notifier).state = index;
  navigationShell.goBranch(index);
},
```

Add the import for `activeTabIndexProvider` — since `app_router.dart` already imports `app_providers.dart` (line 15: `import 'package:catat_cuan/presentation/providers/app_providers.dart';`), no new import is needed. The provider is exported from app_providers.dart via that import.

**Step 3: Watch provider in BudgetListScreen**

In `budget_list_screen.dart`, add a tab-switch detection mechanism. Since `budget_list_screen.dart` already imports `app_providers.dart` (line 7), no new import is needed.

Add a field to track the last seen tab index:
```dart
int _lastSeenTabIndex = 1; // Anggaran tab index
```

In the `build()` method, at the very top of the method body (before the `return FutureBuilder`), add tab-switch detection:
```dart
// Detect tab re-activation: refresh budget data when switching back to Anggaran tab
final activeTabIndex = ref.watch(activeTabIndexProvider);
if (activeTabIndex == 1 && _lastSeenTabIndex != 1) {
  // Switched back to Anggaran tab — schedule a refresh after current build
  WidgetsBinding.instance.addPostFrameCallback((_) {
    _loadBudgets();
  });
}
_lastSeenTabIndex = activeTabIndex;
```

**Why this works:**
- `StatefulShellRoute.indexedStack` preserves all tab widgets in memory
- `ref.watch(activeTabIndexProvider)` causes BudgetListScreen to rebuild whenever the provider changes
- When the provider value becomes 1 (Anggaran) and it was previously not 1, we know the user switched back
- `addPostFrameCallback` ensures `_loadBudgets()` runs after the current build cycle (avoiding setState during build)
- `_loadBudgets()` fetches fresh data and calls `setState()`, which triggers the FutureBuilder to show updated data

**Important constraints:**
- Tab indices: 0=Transaksi, 1=Anggaran, 2=Laporan (from `activeTabs` in app_router.dart)
- Do NOT import from `app_router.dart` in `budget_list_screen.dart` — that would create a circular dependency. Use `app_providers.dart` as the shared location.
- The `_lastSeenTabIndex` initial value is 1 because BudgetListScreen starts as the active tab when the app first navigates to it
