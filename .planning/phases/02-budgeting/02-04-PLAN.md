  <action>
Fix three reactivity problems in budget screens:

**Problem A — Push not awaited (test 4):**
In `budget_list_screen.dart`, find the two places `context.push(...)` is called:
1. Line ~182 in `_buildBudgetCard` onTap: `context.push('${AppRoutes.budgets}/detail?...')`
2. Line ~213 in `_buildEmptyState` onAction: `context.push('${AppRoutes.budgets}/form?...')`

Change BOTH to `await context.push<bool>(...)`. After each, call `setState(() {})` to trigger FutureBuilder re-fire:
```dart
final result = await context.push<bool>('${AppRoutes.budgets}/form?...');
if (result == true) setState(() {});
```

The budget_form_screen.dart already calls `context.pop(true)` on success (line 286), so the result is already wired.

**Problem B — Tab switch doesn't refresh (tests 8, 9):**
The root cause is `StatefulShellRoute.indexedStack` preserves state and FutureBuilder fires once. The cleanest fix without converting to Riverpod AsyncNotifier is to store the future in a field and re-create it in `setState`.

Change the `build` method so that the Future is stored as a field:
```dart
Future<List<BudgetWithSpentEntity>>? _budgetsFuture;

void _loadBudgets() {
  _budgetsFuture = ref.read(getBudgetWithSpentUseCaseProvider)(
    MonthParams(year: _currentMonth.year, month: _currentMonth.month),
  ).then((result) => result.isSuccess ? result.data ?? [] : []);
  setState(() {});
}
```

Then in `initState`, call `_loadBudgets()`. In `build`, use `_budgetsFuture!` in the FutureBuilder. All calls to `setState(() {})` that should trigger a refresh (e.g., after push returns, month changes, RefreshIndicator) should call `_loadBudgets()` instead.

**Problem C — Budget detail screen refresh (test 9):**
In `budget_detail_screen.dart`, apply the same pattern. Store the future in a field `_budgetsFuture` with a `_loadBudgets()` method. The existing `RefreshIndicator` already calls `setState(() {})` — change it to call `_loadBudgets()` instead.

For the detail screen, the data already re-fetches on `setState` because `FutureBuilder.future` is set from a method call. Verify that after navigating to detail → popping back → navigating to detail again shows fresh data. The key: `_loadBudgets()` must create a NEW future each time (not reuse the same future object).
