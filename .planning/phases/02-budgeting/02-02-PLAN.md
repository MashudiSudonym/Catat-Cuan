    - CreateBudgetUseCase: extends `UseCase<BudgetEntity, CreateBudgetParams>` where CreateBudgetParams has categoryId, year, month, amount. Validate amount > 0, month in 1..12, year >= 2020. Return ValidationFailure on invalid. Call BudgetWriteRepository.createBudget.

    **CheckBudgetAlertsUseCase** (core alert logic per D-02/D-03):
    - Input: `BudgetAlertParams { int categoryId, int year, int month }`
    - Logic:
      1. Fetch budget for this category+month via BudgetReadRepository
      2. If no budget exists → return none
      3. Fetch spent amount via BudgetQueryRepository.getBudgetSpentForCategory
      4. Calculate progressPercent = (spent / budget.amount * 100)
      5. Read alert_status fields from budget (warning_shown_at, limit_shown_at, over_shown_at)
      6. Determine highest threshold crossed: >100% > 100% > 75% > none
      7. Check if that threshold's alert was already shown (field is non-null)
      8. If new alert needed: update the alert_status field with current timestamp, return alert type
      9. Return BudgetAlertStatus indicating which alert to show
    - Output: `BudgetAlertResult { BudgetAlertType type, BudgetEntity? budget, double spentAmount }` where BudgetAlertType enum is: none, warning, limit, over
