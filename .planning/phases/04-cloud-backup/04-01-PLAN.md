<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App → Google OAuth | User credentials cross to Google servers; app only receives OAuth tokens |
| App → Google Drive API | Auth tokens sent with each API call; drive.appdata scope limits access to app-only folder |
| FlutterSecureStorage → Device | Tokens stored encrypted in platform keystore (Keychain/EncryptedSharedPreferences) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | S (Spoofing) | Auth tokens | mitigate | Use HTTPS-only for all Google API calls; verify SSL certificates |
| T-04-02 | T (Tampering) | Token storage | mitigate | flutter_secure_storage provides encryption at rest; platform Keychain/EncryptedSharedPreferences |
| T-04-03 | I (Info Disclosure) | Auth tokens at rest | mitigate | Encrypted storage prevents token extraction from device |
| T-04-04 | E (Elevation) | drive.appdata scope | mitigate | Only request drive.appdata scope — no access to user's personal Drive files |
| T-04-05 | D (Denial) | Network errors | accept | Network failures are transient; user can retry. No PII at risk. |
</threat_model>
