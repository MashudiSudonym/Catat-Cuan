Three surgical fixes. Each is a single-line removal or addition:

**Fix 1 — Icon picker double-pop** in `lib/presentation/widgets/category_icon_picker.dart`:
In `CategoryIconPickerDialog.build()`, find the `CategoryIconPicker` widget's `onIconSelected` callback (around lines 145-148):
```dart
onIconSelected: (icon) {
  onIconSelected(icon);
  Navigator.of(context).pop();  // <-- REMOVE THIS LINE
},
```
Remove the `Navigator.of(context).pop();` line (line 148). The `onIconSelected(icon)` call already triggers the `show()` method's callback which does `Navigator.of(context).pop(icon)` — that single pop is correct and closes the dialog. The second pop is the bug that removes the savings goal form screen from the root navigator stack.

**Fix 2 — Color picker double-pop** in `lib/presentation/widgets/category_color_picker.dart`:
In `CategoryColorPickerDialog.build()`, find the `CategoryColorPicker` widget's `onColorSelected` callback (around lines 79-81):
```dart
onColorSelected: (color) {
  onColorSelected(color);
  Navigator.of(context).pop();  // <-- REMOVE THIS LINE
},
```
Remove the `Navigator.of(context).pop();` line (line 81). Same pattern as fix 1 — `onColorSelected(color)` already triggers `show()`'s callback which pops the dialog with the selected color.

**Fix 3 — Stale goal list** in `lib/presentation/screens/savings/savings_goal_form_screen.dart`:
In the `_submit()` method, inside the `if (result.isSuccess)` block (around line 436-445), add `ref.invalidate(savingsGoalsWithProgressProvider);` BEFORE `context.pop(true)`. The block should become:
```dart
if (result.isSuccess) {
  ScaffoldMessenger.of(context).showSnackBar(
    SnackBar(
      content: Text(
        _isEditing ? 'Goal berhasil diperbarui' : 'Goal berhasil dibuat',
      ),
      backgroundColor: AppColors.success,
    ),
  );
  ref.invalidate(savingsGoalsWithProgressProvider);
  context.pop(true);
}
```

This invalidates the `FutureProvider.autoDispose` that backs the savings goal list. Since the list screen lives in a `StatefulShellBranch.indexedStack` (permanently mounted), autoDispose never triggers — explicit invalidation is required to force a fresh fetch after save.
